## Q1. Give the Importance of Agile Process

**Definition:**  
Agile is an **iterative and incremental** software development methodology that focuses on flexibility, collaboration, and customer satisfaction.

**Importance:**

- Delivers working software in **short iterations (sprints)** → faster feedback
- Adapts easily to **changing requirements** → reduces project failure risk

---

## Q2. What is Meant by Extreme Programming (XP)?

**Definition:**  
Extreme Programming (XP) is an **Agile software development framework** that focuses on producing high-quality software through frequent releases and continuous customer involvement.

**Key Points:**

- Uses practices like **pair programming, test-first development, and continuous integration**
- Emphasizes **simplicity, communication, feedback, and courage** as core values

## Q3. Give the Primary Goals in the Design of UML

**Definition:**  
UML (Unified Modeling Language) is a **standardized visual language** for modeling software systems.

**Primary Goals:**

- Provide a **ready-to-use, expressive visual modeling language** applicable to all domains
- Support **independence from specific programming languages and development processes** → ensures universality

## Q4. Differentiate Sequence and Collaboration Diagram

**Definition:**  
Both are **interaction diagrams** in UML that show how objects communicate — but they emphasize different aspects.

|Feature|Sequence Diagram|Collaboration Diagram|
|---|---|---|
|**Focus**|Time ordering of messages|Structural relationship between objects|
|**Shows**|Vertical timeline with lifelines|Objects linked with numbered messages|
|**Best for**|Showing message sequence clearly|Showing object organization|
|**Axis**|Time (top to bottom)|Space (no time axis)|

## Q5. "A System Must Be Loosely Coupled and Highly Cohesive" — Justify

**Loose Coupling:**  
Modules should have **minimum dependency** on each other → a change in one module does not affect others → easier maintenance.

**High Cohesion:**  
Each module should perform **one well-defined task** → all elements inside a module are strongly related → increases readability and reusability.

**Justification:**

- Loose coupling + High cohesion → **modular, maintainable, scalable** software
- Reduces **ripple effect** of changes and improves **fault isolation**

## Q6. State Design Pattern

**Definition:**  
A **Design Pattern** is a reusable, proven solution to a commonly occurring problem in software design — it is a **template**, not a complete code solution.

**Key Points:**

- Classified into 3 types: **Creational, Structural, Behavioral** (by Gang of Four)
- Example: **Singleton pattern** ensures only one instance of a class exists throughout the system

## Q7. Analyse About CASE Tools and Where It Is Used

**Definition:**  
**CASE (Computer-Aided Software Engineering) tools** are software applications that support and automate various activities in the software development lifecycle.

**Where It Is Used:**

- **Upper CASE tools** → used in planning, analysis, and design phases (e.g., drawing DFDs, ER diagrams)
- **Lower CASE tools** → used in coding, testing, and maintenance phases (e.g., code generators, debuggers)

---

## Q8. Illustrate the Impact of Object Orientation in Testing

**Definition:**  
Object-oriented testing involves testing **classes, objects, methods, and their interactions** rather than individual functions.

**Impact:**

- Testing shifts from **function-level** to **class/object-level** → introduces new challenges like testing **inheritance, polymorphism, and encapsulation**
- Requires specialized techniques like **class testing, integration testing of collaborating objects**, and **scenario-based testing**

---

## Q9. What Are the Main Functions of Software Configuration Management (SCM)?

**Definition:**  
SCM is the discipline of **tracking and controlling changes** in software throughout its lifecycle.

**Main Functions:**

- **Version Control** — tracks all versions/revisions of software components (e.g., Git)
- **Change Control, Build Management & Release Management** — controls what changes are approved, builds consistent software, and manages product releases

---

## Q10. List Out the Steps in Project Scheduling

**Definition:**  
Project scheduling is the process of **defining tasks, estimating durations, and arranging them in a sequence** to meet project deadlines.

**Steps:**

1. **Define activities** → identify all tasks from WBS
2. **Sequence activities** → determine order/dependencies
3. **Estimate resources & duration** → assign time and people to each task
4. **Develop & control schedule** → create Gantt chart / CPM network and monitor progress
