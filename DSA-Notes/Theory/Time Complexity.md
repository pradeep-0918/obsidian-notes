## 📌 Definition

**Time Complexity** measures how algorithm runtime grows as input size n increases. We care about asymptotic behavior (large n).

---

## 📊 Common Algorithms by Complexity

### O(1) — Constant
- Array index access
- Hash table lookup/insert
- Stack push/pop

### O(log n) — Logarithmic
- Binary search
- BST insert/search (balanced)
- Heap insert/extract

### O(n) — Linear
- Array scan
- BFS / DFS
- Kadane's algorithm

### O(n log n) — Linearithmic
- Merge sort, Heap sort, Quick sort (avg)
- Building a heap from array

### O(n²) — Quadratic
- Bubble/Selection/Insertion sort
- Brute force 2-sum (no hash)
- DP with n² states

### O(2ⁿ) — Exponential
- Generating all subsets
- Recursive Fibonacci (no memo)
- Exhaustive backtracking

### O(n!) — Factorial
- Generating all permutations
- Traveling Salesman (brute)
