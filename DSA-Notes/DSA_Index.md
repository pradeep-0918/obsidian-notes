## 🧩 Problem-Solving Strategy

1. **Understand** — inputs, outputs, constraints, examples
2. **Identify Pattern** — sliding window? two pointers? DP?
3. **Choose Data Structure** — which fits the access pattern?
4. **Code** — brute force first, then optimize
5. **Analyze** — time & space complexity
6. **Test** — edge cases, empty input, large input

### Common Patterns Quick Reference
| Pattern | When to Use |
|---|---|
| Two Pointers | Sorted arrays, pair sums, palindromes |
| Sliding Window | Subarray/substring optimization |
| Binary Search | Sorted data, search space reduction |
| DFS/BFS | Trees, graphs, path finding |
| Dynamic Programming | Overlapping subproblems, optimal substructure |
| Greedy | Local optimal → global optimal |
| Backtracking | Permutations, combinations, constraints |
| Monotonic Stack | Next greater/smaller element |
| Union Find | Connected components, cycle detection |
| Topological Sort | Dependency ordering, DAGs |
| Two Heaps | Median finding, scheduling |
| Trie | Prefix search, word problems |

| Linked List | O(n) | O(n) | O(1) | O(1) |

| Trie | — | O(m) | O(m) | O(m) |
